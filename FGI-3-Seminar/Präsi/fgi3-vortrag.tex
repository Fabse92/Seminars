\section{CTL Model Checking mit OBDDs}
\begin{frame}
  \frametitle{ Computational Tree Logic }
	CTL Syntax:
	\begin{align*}
		\phi &::= p\,|\,\lnot\phi\,|\,\phi\lor\phi\,|\,EX\phi\,|\,EG\phi\,|\,E[\phi U\phi]\,|.
	\end{align*}
	\medskip
	
	CTL Semantik:
	\begin{itemize}
		\item Wird entlang der möglichen Pfade eines Transitionssystems TS ausgewertet
		\item $TS = (S,R,V)$
		\begin{itemize}
			\item S: Zustandsmenge
			\item R: Übergangsrelation $R \subseteq S \times S$
			\item V: Evaluierungsfunktion $S \rightarrow 2^P$
			%P Menge der atomaren Aussagen
		\end{itemize}
	\end{itemize}
\end{frame}

\begin{frame}
	\frametitle{ labelling algorithm $MC$($\phi$, M) }
	Berechnet die Menge der Zustände in M, in denen $\phi$ gilt:\bigskip
	
	\begin{itemize}
		\item Für atomare Formeln: $MC$($\phi$, M) = V($\phi$)
		\item Für $\phi = \phi_1 \lor \phi_2$: $MC$($\phi$, M) = $MC$($\phi_1$, M) $\cup$ $MC$($\phi_2$, M)
		\item Für $\phi$ = EX$\phi_1$: Die Menge der Zustände, in denen eine Transition in einen Zustand möglich ist, in dem $\phi_1$ gilt
		\item Für EG$\phi_1$ und E[$\phi_1$U$\phi_2$] werden Fixpunkte genutzt
		%Für weitere Details siehe Paper
	\end{itemize}
	%Zur Überprüfung, ob ein Zustand eine Spez. erfüllt, muss man nur schauen, ob Zustand in der Menge enthalten ist
\end{frame}

\begin{frame}
	\frametitle{ OBDDs }
	\begin{itemize}
		\item OBDDs sind Binärbäume, die Boolesche Funktionen repräsentieren
		\item Minimale OBDDs sind eindeutig für äquivalente Boolesche Funktionen
	\end{itemize}

		\caption{OBDDs zur Formel $f=x1\land (x2 \lor x3)$}

\end{frame}

\begin{frame}
	\frametitle{ OBDDs zum Model Checking }
	%Wir wollen wissen, ob Spez. in allen Zuständen im Modell gilt
	Frage: Gilt $M\models\phi$?\medskip
	
	\begin{itemize}
		\item Modell M wird als OBDD dargestellt
		\begin{itemize}
			\item Zustände und Übergangsrelation müssen zu Booleschen Funktionen übersetzt werden
			%das ist notwendig, um Formeln zu verifizieren, die über Pfade Quantifiziert sind
		\end{itemize}
		\item Der \textit{labelling algorithm} wird auf OBDDs umgestellt
		\begin{itemize}
			\item Mengenoperationen müssen für OBDDs definiert werden
		\end{itemize}
		\item Der Vergleich der OBDDs von M und $MC$($\phi$, M) liefert die Antwort auf unsere Frage
	\end{itemize}
\end{frame}

\section{ Interpreted systems }

\begin{frame}
  \frametitle{ Interpreted systems }
  In MCMAS werden Multiagentensysteme als \textit{interpreted systems} modelliert. Sie haben folgende Bestandteile:\bigskip
  \begin{itemize}
  	\item Eine Menge lokaler Zustände
  	\item Eine Menge möglicher Aktionen
  	\item Ein Protokoll
  	%Welche Aktionen sind in welchen Zuständen möglich
  	\item Eine Evolutionsfunktion
  	%welche Zustandsübergänge werden wann vollzogen
  \end{itemize}
\end{frame}

\section{Das Tool MCMAS}
\begin{frame}
	\frametitle{ MCMAS }
	Das Model Checking Tool MCMAS:\bigskip	
	\begin{itemize}
	\item Nutzt OBDDs zum Model Checking auf \textit{interpreted systems}
	\item Erweitert die Konzepte um weitere Logiken
		\begin{itemize}
		\item Bspw. um den \textit{knowledge}-Operator K
		%ein Agent weiß etwas, wenn er es ausschließlich aus seinen lokalen Zuständen schließen kann. Unabhängig von allen anderen Umständen
		\end{itemize}
	\end{itemize}
\end{frame}
